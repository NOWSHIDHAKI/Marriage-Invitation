🚀 5-MINUTE SETUP
═══════════════════════════════════════════════════════════════════════════

STEP 1: Create Supabase Account (2 minutes)
   → Go to https://supabase.com
   → Sign up for free
   → Create a new project named "wedding-rsvp"
   → Wait for project initialization (auto.com)
   → Note: Save your database password somewhere secure

STEP 2: Set Up Database Table (1 minute)
   → In Supabase dashboard, go to SQL Editor
   → Click "New query"
   → Copy the SQL script from SUPABASE_SETUP_GUIDE.md
   → Click "Run"
   → ✅ Your table is created!

STEP 3: Get Your Credentials (1 minute)
   → In Supabase, go to Settings → API
   → Copy your "Project URL" (looks like: https://xxxxx.supabase.co)
   → Copy your "Anon Key" (long string starting with eyJ...)

STEP 4: Update Configuration (1 minute)
   → Open supabase-config.js in this folder
   → Find the top lines with placeholders
   → Paste your credentials there
   → Save the file

✨ DONE! Test your setup:
   → Open invitation.html in your browser
   → Fill out the RSVP form
   → Submit
   → You should see a success message!

⚠️ TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

ISSUE: "Supabase client failed to initialize"
   → Check that supabase-config.js is in the same folder
   → Verify SUPABASE_URL and SUPABASE_KEY in the file
   → Open browser console (F12) for specific error

ISSUE: Form submits but no success message
   → Check browser console for errors (F12 → Console)
   → Verify your credentials are pasted correctly
   → Make sure you've run the SQL script in Supabase

ISSUE: Admin dashboard shows "Could not load RSVP data"
   → Verify supabase-config.js has correct credentials
   → Check that the table "rsvp_responses" exists in Supabase
   → Try refreshing the page

ISSUE: Data not appearing in Supabase
   → Log in to Supabase dashboard
   → Go to Table Editor → rsvp_responses
   → Check if table exists and has data
   → Verify RLS policies allow inserts
